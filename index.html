Frequently Asked Questions




1. What is the difference between var, let, and const?
Answer: var works everywhere in a function but can be messy because we can reuse it and it gets set to undefined before it runs. let only works inside the block ({}), we can change it but not redeclare it. const is like let, but we can’t change it after setting it.

2. What is the difference between map(), forEach(), and filter()?
Answer: map() goes through each item in an array and makes a new array with the changed values. forEach() goes through the array but doesn’t give us a new one, it just runs a function for each item. filter() gives us a new array with only the items that pass a condition

3. Explain arrow functions and how they are different from regular functions.
Answer: Arrow functions are shorter and simpler, and they don’t create their own this — they use the this from where they were written. Regular functions create their own this and are more flexible when we need that behavior.

4. How JavaScript Promises work?
Answer: JavaScript Promise are like a guarantee that something will finish later. It starts as pending, then either works (fulfilled) or fails (rejected). We use then() when it works, catch() if it fails, and finally() to run something no matter what happens.

5. How closures work in JavaScript?
Answer: A closure is when a function remembers the variables around it, even after that part of the code is done running. It’s like a function with a memory, helpful for keeping data safe or making small functions that "remember" stuff.
